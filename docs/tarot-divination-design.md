# 塔罗牌占卜游戏设计方案

## 1. 产品定位

本项目不是传统意义上的“塔罗卡牌游戏”，而是一个以塔罗占卜为核心体验的互动式命理产品。它需要让玩家感到：

- 抽牌过程不是随机点按钮，而是一次有仪式感的占卜行为。
- 结果解析不是泛泛的安慰文案，而是能回应玩家当前问题。
- 牌面解释具有一致规则，不会为了迎合玩家而随意发挥。
- 情绪反馈是辅助，不替代占卜判断。

核心目标是同时满足两类需求：

1. **准确感**：玩家认为结果和自己的处境、问题、情绪有明显对应关系。
2. **情绪承接**：即使结果偏负面，也能给出可行动、可接受的解释，不制造恐慌。

设计原则：

- 先确定问题类别，再抽牌，再解析，不能直接套模板。
- 牌位、牌义、正逆位、主题语境必须共同参与解释。
- 结果允许不确定，但不允许空泛。
- 对玩家做建议，不做绝对预言。
- 对高风险问题做温和边界处理，例如医疗、法律、投资、生死等。

## 2. 核心体验循环

完整占卜流程如下：

```text
选择占卜主题
-> 选择具体问题类别
-> 选择对象、时间范围和关注重点
-> 系统生成标准化占卜问题
-> 选择推荐牌阵
-> 玩家洗牌
-> 玩家切牌
-> 玩家抽牌
-> 逐张翻牌
-> 单牌解释
-> 牌组关系分析
-> 综合结论
-> 行动建议与情绪反馈
-> 保存结果
```

玩家感知重点：

- 抽牌前，系统要让玩家“聚焦问题”。
- 抽牌中，玩家要感觉自己参与了随机结果的形成。
- 翻牌时，要有短暂停顿和期待感。
- 解读时，要先给判断，再解释理由，最后给建议。

## 3. 占卜主题

第一版建议支持 6 个主题。

| 主题 | 适用问题 | 解析重点 |
| --- | --- | --- |
| 今日指引 | 今天要注意什么、今天适合做什么 | 当日状态、提醒、行动建议 |
| 感情关系 | 对方怎么想、关系会怎样发展 | 双方状态、隐藏阻力、沟通方式 |
| 事业学业 | 工作机会、考试、项目推进 | 资源、压力、执行路径、风险 |
| 选择判断 | A 和 B 怎么选 | 两个方向的利弊、关键变量 |
| 内在状态 | 我为什么焦虑、我真正想要什么 | 潜意识、情绪来源、自我认知 |
| 近期趋势 | 某件事接下来会怎样 | 当前动能、阻碍、短期走向 |

主题决定：

- 可选牌阵。
- 牌位名称。
- 牌义解释侧重点。
- 结果语气。
- 风险提示规则。

## 4. 问题类别系统

本项目不提供自由文本问题输入。所有问题都通过类别、选项和参数组合生成，避免程序误解玩家意图，也避免高风险、绝对化或过度私人化的问题进入占卜流程。

这样做的好处：

- 玩家不需要组织语言，进入占卜更快。
- 系统不需要依赖 NLP 判断问题质量。
- 所有问题天然符合塔罗解释边界。
- 解析模板能稳定匹配牌阵、牌位和主题。
- 更容易控制风险问题和商业化内容范围。

### 4.1 问题选择流程

玩家通过 3-4 步选择生成标准化问题。

```text
选择主题
-> 选择问题类别
-> 选择时间范围
-> 选择对象或关注重点
-> 生成标准化占卜问题
-> 玩家确认并开始洗牌
```

示例：

```text
主题：感情关系
问题类别：这段关系接下来会怎样
时间范围：未来一个月
对象：某个我在意的人
生成问题：我和这个人在未来一个月的关系发展趋势是什么？
```

### 4.2 问题类别表

每个主题下配置固定问题类别。

| 主题 | 问题类别 | 生成问题 |
| --- | --- | --- |
| 今日指引 | 今天最需要注意什么 | 今天我最需要注意的状态和行动是什么？ |
| 今日指引 | 今天适合推进什么 | 今天我适合把精力放在哪个方向？ |
| 感情关系 | 关系接下来会怎样 | 我和这个人在未来一段时间的关系发展趋势是什么？ |
| 感情关系 | 对方目前的状态 | 对方目前在这段关系中表现出的状态是什么？ |
| 感情关系 | 我该不该继续投入 | 我是否适合继续在这段关系中投入精力？ |
| 事业学业 | 当前机会是否值得推进 | 这个机会在近期是否值得继续推进？ |
| 事业学业 | 当前阻力在哪里 | 当前事业或学业中的主要阻力是什么？ |
| 事业学业 | 下一步行动建议 | 我接下来最适合采取什么行动？ |
| 选择判断 | A 和 B 哪个更适合 | 在当前条件下，两个选择各自的趋势和风险是什么？ |
| 内在状态 | 我真正的情绪来源 | 我当前情绪背后真正需要被看见的是什么？ |
| 内在状态 | 我需要放下什么 | 我现在最需要放下或调整的是什么？ |
| 近期趋势 | 这件事接下来会怎样 | 这件事在近期的发展趋势是什么？ |
| 近期趋势 | 当前最关键变量 | 影响这件事发展的关键变量是什么？ |

### 4.3 问题参数

问题类别之外，玩家可以选择少量参数，让问题更贴近个人处境，但不开放自由输入。

通用参数：

```text
时间范围：今天 / 未来 7 天 / 未来一个月 / 未来三个月
关注重点：状态 / 阻力 / 趋势 / 建议 / 选择
结果语气：直接一点 / 温和一点
```

感情关系参数：

```text
对象关系：暧昧对象 / 恋人 / 前任 / 伴侣 / 暗恋对象 / 不想标记
关系阶段：刚认识 / 发展中 / 冷淡期 / 争执后 / 分开后 / 不确定
```

事业学业参数：

```text
场景：工作 / 项目 / 考试 / 求职 / 合作 / 学习计划
目标：推进 / 稳定 / 转换方向 / 判断机会 / 找出阻力
```

选择判断参数：

```text
选择类型：关系选择 / 工作选择 / 学业选择 / 生活选择 / 其他选择
决策状态：倾向 A / 倾向 B / 都不确定 / 时间紧迫
```

### 4.4 标准化问题生成

系统通过模板把选项组合成标准化问题。

配置示例：

```json
{
  "id": "love_relationship_trend",
  "topic": "love",
  "label": "关系接下来会怎样",
  "requiredParams": ["timeRange", "relationshipStage"],
  "defaultSpread": "relationship_5",
  "questionTemplate": "我和这个人在{timeRange}的关系发展趋势是什么？",
  "interpretationFocus": ["双方状态", "隐藏阻力", "近期发展", "行动建议"],
  "blocked": false
}
```

生成结果：

```json
{
  "questionId": "love_relationship_trend",
  "topic": "love",
  "displayQuestion": "我和这个人在未来一个月的关系发展趋势是什么？",
  "timeRange": "未来一个月",
  "defaultSpread": "relationship_5",
  "interpretationFocus": ["双方状态", "隐藏阻力", "近期发展", "行动建议"]
}
```

解析引擎只接收 `questionId` 和标准化后的 `displayQuestion`，不接收玩家自由文本。

### 4.5 问题质量如何保证

由于所有问题来自配置表，问题质量通过设计时控制，而不是运行时理解。

保证方式：

```text
每个问题类别必须绑定主题
每个问题类别必须绑定默认牌阵
每个问题类别必须绑定解释重点
每个问题类别必须绑定允许的时间范围
每个问题类别必须经过风险审查
每个问题类别不能包含绝对预言词
```

禁止出现的问题类型：

```text
他一定会回来吗
我什么时候会死
我该不该停药
我投资能不能稳赚
他心里到底有没有我
彩票号码是什么
```

对应替代类别：

| 禁止问题 | 替代类别 |
| --- | --- |
| 他一定会回来吗 | 关系接下来会怎样 |
| 他心里到底有没有我 | 对方目前的状态 |
| 我该不该停药 | 当前情绪支持 |
| 我投资能不能稳赚 | 当前机会的风险与阻力 |
| 彩票号码是什么 | 不提供替代，占卜不支持 |

### 4.6 类别选择界面

界面不出现大段输入框，只展示可点击选项。

一级：主题选择

```text
今日指引
感情关系
事业学业
选择判断
内在状态
近期趋势
```

二级：问题类别选择

```text
关系接下来会怎样
对方目前的状态
我该不该继续投入
这段关系的阻力是什么
```

三级：参数选择

```text
未来 7 天
未来一个月
未来三个月
```

确认页：

```text
本次占卜将回应：
“我和这个人在未来一个月的关系发展趋势是什么？”
```

玩家可选：

```text
开始洗牌
重新选择
```

### 4.7 扩展方式

后续新增问题不需要改解析系统，只需要新增配置。

新增问题类别需要填写：

```text
questionId
所属主题
展示名称
标准化问题模板
允许时间范围
推荐牌阵
解释重点
风险等级
是否付费
是否可每日重复
```

这让问题系统可以像内容库一样运营，而不是每次都依赖模型理解。

## 5. 牌阵设计

### 5.1 单张指引

适用主题：

- 今日指引
- 内在状态

牌位：

```text
当前最重要的提醒
```

输出结构：

```text
当前状态
牌面提醒
今日建议
```

### 5.2 三张趋势牌阵

适用主题：

- 感情关系
- 事业学业
- 近期趋势

牌位：

```text
1. 当前状态
2. 隐藏影响
3. 近期趋势
```

解释重点：

- 第一张判断玩家现在所处的位置。
- 第二张解释真正影响结果的因素。
- 第三张只判断近期趋势，不做长期绝对预言。

### 5.3 感情关系牌阵

适用主题：

- 感情关系

牌位：

```text
1. 你的真实状态
2. 对方的表现或态度
3. 关系中的阻力
4. 近期发展
5. 建议
```

该牌阵必须区分：

- 对方真实想法。
- 对方外在表现。
- 玩家自己的投射和期待。

不能把“对方态度”直接写成绝对心理读心。更稳妥的表达是：

```text
这张牌更像是在说明对方目前表现出的状态，而不是证明对方内心一定如此。
```

### 5.4 选择牌阵

适用主题：

- 选择判断

牌位：

```text
1. 选项 A 的优势
2. 选项 A 的隐患
3. 选项 B 的优势
4. 选项 B 的隐患
5. 你真正需要重视的标准
```

输出不直接替玩家做决定，而是给出倾向：

```text
更适合推进 A
更适合观察 B
两个选择都不成熟
当前不适合立刻决定
```

### 5.5 深度十字牌阵

适用主题：

- 近期趋势
- 内在状态
- 复杂关系

牌位：

```text
1. 问题核心
2. 当前阻碍
3. 表层条件
4. 潜在动机
5. 过去影响
6. 近期趋势
```

第一版可以作为解锁牌阵或付费牌阵，不建议默认开放。

## 6. 抽牌规则

### 6.1 基础牌库

使用标准韦特塔罗结构：

```text
大阿尔卡那：22 张
小阿尔卡那：56 张
总计：78 张
```

MVP 可以先实现 22 张大阿尔卡那，但系统结构要按 78 张设计。

### 6.2 随机规则

基础规则：

```text
同一次占卜中无放回抽牌
同一次占卜中不能重复出现同一张牌
每张牌独立计算正逆位
默认正位概率：70%
默认逆位概率：30%
```

不做新手保护，不因玩家是新用户而降低逆位、负面牌或强冲突组合的出现概率。情绪体验由解析表达和行动建议承接，不通过修改抽牌结果实现。

### 6.3 随机种子

为了增强仪式感，随机种子由多项行为共同生成：

```text
玩家 ID
当前日期
问题文本 hash
洗牌拖动轨迹
切牌位置
抽牌顺序
设备时间扰动
```

今日指引使用日期级固定种子：

```text
同一玩家同一天同一主题只生成一个正式结果
```

再次抽取时提示：

```text
你今天已经问过这个问题。可以重新查看结果，或开启一次“再次追问”。
```

再次追问的结果需要标记为：

```text
回声占卜
```

它的解释权重低于第一次结果。

## 7. 抽牌交互动画

### 7.1 类别确认阶段

界面表现：

- 中央显示牌桌。
- 牌堆处于暗色静止状态。
- 玩家完成主题、问题类别和参数选择后，牌堆出现微弱呼吸光。
- 界面展示本次生成的标准化占卜问题。

文案：

```text
本次占卜将回应这个问题。确认后，请专注于它并开始洗牌。
```

### 7.2 洗牌阶段

交互：

```text
长按牌堆开始洗牌
左右拖动改变牌堆散开方向
拖动越充分，洗牌完成度越高
```

动画：

- 卡牌以扇形展开。
- 部分卡牌旋转 5-20 度。
- 牌背纹样产生微光。
- 洗牌完成时牌堆自动收束。

完成条件：

```text
长按时间 >= 1.5 秒
拖动距离 >= 300 px
或总洗牌时长 >= 4 秒
```

### 7.3 切牌阶段

交互：

```text
玩家在牌堆上横向滑动
牌堆被切成两叠
玩家选择其中一叠置顶
```

设计目的：

- 增强“由我决定”的参与感。
- 将切牌位置纳入随机种子。

### 7.4 抽牌阶段

展示方式：

```text
根据牌阵生成 9-15 张牌背
玩家从中依次选择所需数量的牌
已选牌飞入牌阵槽位
```

抽牌反馈：

- 点击牌背后先轻微上浮。
- 卡牌边缘闪烁。
- 飞向对应牌位。
- 牌位出现短暂光圈。

### 7.5 翻牌阶段

规则：

```text
必须逐张翻牌
不能一次性全部打开
每张牌翻开后展示牌名、正逆位、关键词
```

动画节奏：

```text
点击
-> 牌面抬起
-> Y 轴翻转 180 度
-> 90 度时替换牌图
-> 落回牌位
-> 显示关键词
```

负面牌翻开时不使用惊吓音效，避免制造不必要压力。

## 8. 牌面数据结构

每张牌需要支持多主题解释，而不是只有一个通用牌义。

```json
{
  "id": "major_06",
  "name": "恋人",
  "arcana": "major",
  "number": 6,
  "element": "air",
  "baseEnergy": 1,
  "upright": {
    "keywords": ["选择", "吸引", "价值一致", "关系连接"],
    "coreMeaning": "这张牌代表关系、选择，以及内心价值的确认。"
  },
  "reversed": {
    "keywords": ["犹豫", "错位", "逃避选择", "关系失衡"],
    "coreMeaning": "这张牌逆位时，通常说明关系或选择中存在不一致与回避。"
  },
  "topicMeanings": {
    "love": {
      "upright": "关系中存在真实吸引，但也需要面对选择与承诺。",
      "reversed": "双方的期待可能并不一致，或有人正在回避明确表态。"
    },
    "career": {
      "upright": "你需要选择更符合长期价值的合作或路径。",
      "reversed": "当前决策可能被短期情绪或他人期待干扰。"
    },
    "inner": {
      "upright": "你正在寻找与真实自我更一致的答案。",
      "reversed": "你可能在迎合外界，而忽略了自己的真实判断。"
    }
  },
  "advice": {
    "upright": "做选择前，先确认什么对你真正重要。",
    "reversed": "不要急着给出承诺，先看清双方是否站在同一个方向。"
  },
  "riskLevel": "medium"
}
```

关键字段说明：

- `baseEnergy`：用于计算整体倾向。
- `topicMeanings`：保证不同主题下解释不跑偏。
- `riskLevel`：用于判断是否需要温和表达。
- `advice`：保证结果最终落到行动建议。

## 9. 解析引擎

解析引擎需要从“抽到什么牌”升级为“在什么问题、什么牌位、什么组合下抽到这张牌”。

### 9.1 输入

```json
{
  "topic": "love",
  "questionId": "love_relationship_trend",
  "displayQuestion": "我和这个人在未来一个月的关系发展趋势是什么？",
  "params": {
    "timeRange": "未来一个月",
    "relationshipStage": "发展中"
  },
  "spread": "relationship_5",
  "cards": [
    {
      "position": "你的真实状态",
      "cardId": "major_06",
      "orientation": "upright"
    },
    {
      "position": "对方的表现或态度",
      "cardId": "major_18",
      "orientation": "reversed"
    }
  ]
}
```

### 9.2 解析层级

解析分 5 层生成。

#### 第一层：问题复述

先确认系统正在回答什么。

```text
这次牌阵回应的是：你和 A 在未来一个月内是否还有继续发展的空间。
```

#### 第二层：单牌解释

每张牌按牌位解释。

```text
在“你的真实状态”位置出现恋人正位，说明你对这段关系并非只是观望，而是已经在认真衡量它是否值得投入。
```

#### 第三层：牌组关系

分析牌之间的互相支持或冲突。

```text
恋人正位与月亮逆位同时出现，说明关系里有吸引，也有不确定感正在下降。问题不一定是没有感情，而是双方是否愿意把模糊的部分说清楚。
```

#### 第四层：综合判断

必须给出明确倾向。

可选结论类型：

```text
可以推进
谨慎推进
适合观察
暂缓行动
建议放下
信息不足
```

示例：

```text
综合来看，这组牌更接近“谨慎推进”。关系并非完全停滞，但它需要一次更清晰的沟通来降低误解。
```

#### 第五层：行动建议

建议必须具体到时间、行为或判断标准。

```text
未来 7 天内可以发起一次轻量沟通，但不要直接逼问关系定义。更适合从近期状态、共同安排或一个具体话题切入。如果对方仍持续回避具体回应，再考虑降低投入。
```

### 9.3 准确感来源

玩家感到准确，通常来自 5 个因素：

1. 问题被准确复述。
2. 牌位解释和问题场景有关。
3. 解析指出矛盾，而不只说好话。
4. 结论有明确倾向。
5. 建议可执行。

因此结果文本必须避免：

```text
一切皆有可能
相信自己
顺其自然
宇宙会给你答案
```

除非这类表达后面跟着具体解释和行动建议。

## 10. 情绪反馈设计

情绪反馈不是把负面牌说成正面，而是让玩家能承受结果。

### 10.1 负面结果表达原则

不要说：

```text
这段关系没有希望。
```

建议说：

```text
这组牌不支持你在短期内继续高强度投入。它更像是在提醒你，对方目前没有给出足够稳定的回应。
```

不要说：

```text
你会失败。
```

建议说：

```text
这组牌显示当前阻力偏强，如果继续推进，需要先补足准备、资源或沟通条件。
```

### 10.2 情绪承接模块

每次解析结尾增加一段情绪承接，但不能喧宾夺主。

结构：

```text
承认感受
降低灾难化
给出下一步
```

示例：

```text
如果你看到这个结果后有些失落，这是正常的。它并不是在否定你的感受，而是在提醒你把注意力从猜测转回可确认的行动上。
```

### 10.3 高风险问题处理

以下问题不能按普通占卜输出判断：

- 医疗诊断。
- 是否停药。
- 自伤或伤害他人。
- 法律诉讼胜负。
- 高风险投资。
- 生死预测。

处理方式：

```text
不直接占卜结论
承接情绪
建议寻求专业帮助
可提供“我现在需要注意什么情绪和行动”类型的辅助占卜
```

示例：

```text
这个问题不适合用塔罗来替代专业判断。你可以把问题改成：“面对这件事，我现在最需要稳定的是什么？”系统可以为你做一次情绪和行动层面的指引。
```

## 12. 历史记录与复盘

为了增强“不是一次性娱乐”的感觉，系统需要保存占卜记录。

记录字段：

```json
{
  "id": "reading_001",
  "createdAt": "2026-06-23T20:00:00+08:00",
  "topic": "love",
  "question": "我和 A 未来一个月关系会怎样？",
  "spread": "relationship_5",
  "cards": [],
  "summary": "谨慎推进",
  "advice": "未来 7 天内适合轻量沟通。",
  "playerNote": "",
  "followUpAt": "2026-06-30"
}
```

复盘功能：

```text
占卜后 7 天提醒玩家回看
玩家可以标记：准确 / 部分准确 / 不准确
玩家可以写下实际发生了什么
```

复盘数据用途：

- 调整文案模板。
- 分析哪些主题准确感更强。
- 优化牌义解释。
- 增强玩家长期信任。

不要把玩家反馈用于直接证明“塔罗一定准确”，而是用于优化解释质量。

## 15. 关键体验验收标准

占卜体验需要满足以下标准：

1. 玩家能在 10 秒内理解自己要做什么。
2. 玩家能感到抽牌过程由自己参与。
3. 每次结果都明确回应玩家的问题。
4. 结果中至少有一处具体判断，而不是全程模糊表达。
5. 负面结果不会造成强烈恐慌。
6. 行动建议具体到可执行行为。
7. 玩家可以保存结果并在之后复盘。
8. 同一天重复追问不会破坏可信度。

## 16. 示例结果

问题：

```text
我和 A 未来一个月关系会怎样？
```

牌阵：

```text
三张趋势牌阵
```

抽牌：

```text
当前状态：恋人 正位
隐藏影响：月亮 逆位
近期趋势：节制 正位
```

解析：

```text
这次牌阵回应的是：你和 A 在未来一个月内是否还有继续发展的空间。

在“当前状态”位置出现恋人正位，说明这段关系中存在真实吸引，你并不是单纯好奇，而是在认真衡量它是否值得继续投入。

在“隐藏影响”位置出现月亮逆位，说明你们之间的不确定感正在下降，但仍有一些没有说清楚的部分。它更像是误解、猜测或信息不完整，而不是单纯没有感情。

在“近期趋势”位置出现节制正位，说明未来一个月的关键词不是快速推进，而是缓慢修复、试探靠近和重新找到合适的相处节奏。

综合来看，这组牌更接近“谨慎推进”。它不指向立刻确定关系，也不指向彻底结束。真正关键的是，你们能否用一次稳定、低压力的沟通，把模糊的部分说清楚。

建议你未来 7 天内可以发起一次轻量沟通，但不要直接逼问关系定义。更适合从近期状态、共同安排或一个具体话题切入。如果对方仍持续回避具体回应，再考虑降低自己的投入。

如果你看到这个结果后仍然不安，这是正常的。牌面并不是要求你立刻做决定，而是在提醒你：先看对方是否愿意给出更稳定的回应，再决定自己要不要继续投入。
```

## 17. 设计结论

这个塔罗占卜系统的核心竞争力不在于牌多，也不在于动画复杂，而在于：

- 问题处理足够认真。
- 抽牌过程足够可信。
- 牌义解释足够稳定。
- 综合判断足够具体。
- 情绪反馈足够克制。

只要这五点成立，玩家就会更容易把它视为一次真正的占卜，而不是一套随机文案游戏。
